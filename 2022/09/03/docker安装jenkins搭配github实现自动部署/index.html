docker安装jenkins搭配github实现自动部署

使用docker命令 docker pull jenkins/jenkins:lts 下载jenkins。

使用docker下载jenkins后，现在开始启动容器。

使用http://{IP}:8888/访问jenkins进行初始化配置。

这里选择安装推荐的插件

下载完成后，会然你创建应该用户

点击保存并完成后一直点击下一步后进入这个页面就已经是安装完成了

在docker的Jenkins容器中安装jdk8,jdk下载链接

安装JDK前先把默认的jdk删除。

下载完成后使用一下步骤：

1.进入容器 docker exec -u root -it (Jenkins)容器ID /bin/bash

2.建立容器系统的JAVA_HOME目录 mkdir /usr/lib/jvm

3.复制gz文件到容器中的/usr/lib/jvm目录下 docker cp gz文件路径 容器ID:/usr/lib/jvm

4.在容器中解压gz文件 tar -zxvf jdk-8u211-linux-x64.tar.gz(你的jdk压缩文件地址)

5.设置环境变量在Jenkins容器中使用vim ~/.bashrc命令

6.使得环境变量马上生效 source ~/.bashrc

7.验证JDK是否安装成功 java -version

开始安装maven仓库，下载地址

下载完成后使用一下步骤：

1.复制maven安装文件到容器中的/usr/local目录下 docker cp maven安装文件路径 容器ID:/usr/local

2.进入容器 docker exec -u root -it (Jenkins)容器ID /bin/bash

3.解压/usr/local目录下的tar -zxvf apache-maven-3.8.4-bin.tar.gz文件

4.配置maven仓库，设置阿里镜像仓库，一定要配置一下，国内的下载jar快些，首先进入cd apache-maven-3.8.4目录，创建仓库存储目录，

5.进入cd conf目录，编辑 vi settings.xml文件，找到mirror 加上阿里的仓库配置，配置完成保存退出。如下图：

6.配置maven环境变量，编辑：vi /etc/profile或者~/.bashrc 文件，翻到最后一行加上 export MAVEN_HOME=/usr/local/apache-maven-3.8.4  export PATH=PATH:PATH:PATH:MAVEN_HOME/bin  保存退出，如下图：

之后使用 source /etc/profile或者~/.bashrc使文件生效，使用mvn -v测试一下是否安装成功

进入到Jenkins界面，新建任务。

选择构建一个maven项目，输入一个任务名点击确定。

现在已经是生成一个Jenkins任务了，需要进行配置一下。先配置一下全局工具配置，在Jenkins主界面选择系统配置->全局工具配置。

全局工具配置就可以啦。现在配置一下我们新建的任务里面的配置，在Jenkins主界面选择我们要配置的任务->进入任务管理界面选择配置。

构建触发器这里我们需要安装一个插件Generic Webhook Trigger，勾择红框中的这三个。 构建环境选择Inspect build log for published Gradle build scans 。

构建触发器Generic Webhook Trigger这里需要配置一下，http://{ip:端口}/generic-webhook-trigger/invoke?token=demo是触发器的地址到时候需要配置到github中，我这里token设置的是demo。

构建触发器配置好后，进入github官网登录后，进入对应的Jenkins配置github项目的项目中选择Settings设置->选择Webhooks菜单->点击Add webhook新建一个当有代码提交到github后推送数据到某个地址，Payload URL填入触发器的地址。

最后一步配置一下构建时的打包命令和构建成功后的需要执行的步骤命令。

现在我们构建测试一下后没有问题就成功了。

需要注意的问题：因为我们是使用docker安装的jenkins在进行项目构建的时候可能会遇到jenkins没有权限的问题，要改一下config.v2.json这个配置文件把user:jenkins改成user:root。
